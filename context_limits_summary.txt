# Context Limits Summary for main.py

## Key Functionalities
1. **Photometry Pipeline**: Handles photometry operations for CCD/NIR images.
2. **Inputs**:
   - FITS file (`-f`)
   - YAML configuration file (`-c`)
   - Optional `-temp` flag for preparing a template image.
3. **Setup**:
   - Working directory for output files.
   - Logging for operations and errors.
   - Metadata extraction (MJD, gain, read noise, saturation level, exposure time, filter, pixel scale).
4. **WCS Handling**:
   - Validates and solves WCS information.
   - Refines WCS solution.
   - Persists updated WCS information.
5. **Preprocessing**:
   - Cosmic-ray removal.
   - Background modeling and subtraction.
   - Image trimming and cropping.
   - Optional north-up reprojection.
6. **Sources**:
   - Builds or downloads a catalog of reference sources.
   - Measures FWHM.
   - Determines optimum aperture.
   - Performs aperture or ePSF+PSF photometry.
7. **Calibration**:
   - Fits zeropoints.
   - Writes calibration information to FITS headers.
8. **Template Subtraction**:
   - Aligns science image with template image.
   - Optionally uses ZOGY algorithm.
   - Applies background correction.
9. **Target Measurement**:
   - Measures target at expected coordinates.
   - Calculates SNR and detection limits.
   - Saves updated FITS files and CSV outputs.

## Key Limitations
1. **Dependencies**: Requires specific tools like SExtractor, SCAMP, and SWarp for advanced features. If these are not found, the pipeline falls back to alternative methods.
2. **Configuration**: Relies heavily on the YAML configuration file for parameters. Missing essential parameters (like target information) will halt the pipeline.
3. **WCS Validation**: Assumes the WCS information in the FITS header is valid. If not, it may lead to incorrect photometry.
4. **Target Information**: Requires at least one of `target_name`, `target_ra`, or `target_dec`. Missing all three will terminate the script.
5. **File Handling**: If the output file already exists and `restart=False`, the script skips processing.
6. **Template Preparation**: When preparing a template, it assumes the template file has the necessary headers (TELESCOP, INSTRUME, FILTER). Missing headers can cause issues.
7. **Image Processing**: Assumes the image data is in a format compatible with the pipeline (e.g., integer data is promoted to float32).
